# WorkOutApp 要件定義書

## 1. プロジェクト概要

### 1.1 アプリ名
**WorkOutApp**

### 1.2 コンセプト
パートナーにバレずにメッセージを送り合えるサービス。
表向きはヘルスケア/フィットネスアプリとして偽装し、秘密のメッセージング機能を提供する。

### 1.3 ターゲットユーザー
束縛の強いパートナーを持つユーザーで、プライベートなコミュニケーション手段を必要とする人

## 3. 認証システム

### 3.1 二重パスワード認証

ユーザーは登録時に2組の認証情報を設定する：

| 種類 | 用途 |
|------|------|
| ダミー認証情報 | パートナーに見せても問題ない偽の認証 |
| 本命認証情報 | 実際のメッセージング機能にアクセス |

### 3.2 認証フロー

```
ログイン画面
    │
    ├─ ダミーパスワード入力 → 偽装ヘルスケア画面（固定サンプルデータ）
    │
    └─ 本命パスワード入力 → 外部ブラウザでWebアプリ起動
```

### 3.3 登録方法
- iOSアプリから登録可能
- Webアプリから登録可能

### 3.4 認証情報の構成

| 項目 | ダミー | 本命 |
|------|--------|------|
| メールアドレス | 必須 | 必須 |
| パスワード | 必須 | 必須 |

## 4. iOSアプリ機能

### 4.1 外観・偽装

| 項目 | 仕様 |
|------|------|
| アプリアイコン | iPhoneのヘルスケアアプリ風のデザイン |
| アプリ名（ホーム画面表示） | 「ヘルスケア」または類似名 |
| UI | 設定画面のみ操作可能に見える仕様 |

### 4.2 ダミー画面（ダミーパスワードでログイン時）

固定のサンプル運動データを表示：
- 歩数
- 運動時間
- 消費カロリー
- 週間/月間グラフ（静的データ）

### 4.3 本命画面（本命パスワードでログイン時）

外部ブラウザ（Safari）でWebアプリを起動
- 将来的に隠し操作（特定箇所を長押しなど）での遷移も検討

### 4.4 プッシュ通知

| 項目 | 仕様 |
|------|------|
| 通知メッセージ | 「アプリのアップデートを行ってください」 |
| 通知タップ時 | アプリが開き、ダミー画面を表示 |
| 実際の意味 | 新着メッセージあり |

### 4.5 スクリーンショット防止
- アプリ内でのスクリーンショットを防止する機能を実装

---

## 5. Webアプリ機能

### 5.1 メッセージング

#### 基本仕様
| 項目 | 仕様 |
|------|------|
| メッセージ形式 | 1対1のみ |
| UIデザイン | LINEをトレース |
| 既読機能 | あり |
| メッセージ検索 | あり |

#### メッセージタイプ
| タイプ | 無料ユーザー | 課金ユーザー |
|--------|--------------|--------------|
| テキスト | ○（月10回まで） | ○（無制限） |
| 画像 | × | ○ |
| 動画 | × | ○ |

#### メッセージ受信
- 無料/課金問わず**無制限**

#### メッセージ削除・送信取消
- **課金ユーザーのみ**利用可能

### 5.2 友達機能

#### 友達追加フロー
```
1. 相手のユーザーIDを入力
2. 友達申請を送信
3. 相手が承認
4. メッセージのやり取りが可能に
```

#### ブロック機能
- 特定ユーザーをブロック可能
- ブロックしたユーザーからのメッセージは受信しない

### 5.3 ユーザープロフィール

| 項目 | 必須/任意 |
|------|----------|
| ユーザーID | 必須（自動生成または任意設定） |
| ニックネーム | 任意 |
| プロフィール画像 | 任意 |

### 5.4 アカウント管理

#### アカウント削除
- 自分のアカウントを削除可能
- 削除時、相手とのトーク履歴も**完全削除**される
- Appleガイドライン準拠（必須機能）

#### 緊急全データ削除（パニックボタン）
- **課金ユーザーのみ**
- ワンタップで全データ（アカウント、メッセージ、友達関係）を即座に削除

## 7. セキュリティ

### 7.1 暗号化

| 項目 | 初期リリース | 将来対応 |
|------|--------------|----------|
| 通信の暗号化 | HTTPS（SSL/TLS） | - |
| データベース暗号化 | Firebaseデフォルト | - |
| E2EE（エンドツーエンド暗号化） | なし | 検討 |

### 7.2 スクリーンショット防止
- iOSアプリ：実装する
- Webアプリ：技術的制限あり（ベストエフォート）

### 7.3 その他
- 生体認証（Face ID / Touch ID）：**初期リリースでは対応しない**
- 通報機能：**実装しない**

## 8. データベース設計（Firestore）

### 8.1 コレクション構成

```
users/
  └── {userId}/
        ├── email: string
        ├── dummyEmail: string
        ├── nickname: string
        ├── profileImage: string (URL)
        ├── isPremium: boolean
        ├── premiumExpiry: timestamp
        ├── messageCount: number (今月の送信数)
        ├── messageCountResetDate: timestamp
        ├── createdAt: timestamp
        └── updatedAt: timestamp

friendships/
  └── {friendshipId}/
        ├── requesterId: string
        ├── receiverId: string
        ├── status: string (pending/accepted/blocked)
        ├── createdAt: timestamp
        └── updatedAt: timestamp

conversations/
  └── {conversationId}/
        ├── participants: array<string>
        ├── lastMessage: string
        ├── lastMessageAt: timestamp
        ├── createdAt: timestamp
        └── messages/ (サブコレクション)
              └── {messageId}/
                    ├── senderId: string
                    ├── type: string (text/image/video)
                    ├── content: string
                    ├── mediaUrl: string (optional)
                    ├── isRead: boolean
                    ├── isDeleted: boolean
                    ├── createdAt: timestamp
                    └── updatedAt: timestamp

fcmTokens/
  └── {userId}/
        └── token: string
```

## 12. 非機能要件

### 12.1 パフォーマンス
- メッセージ送受信は1秒以内
- アプリ起動は3秒以内

### 12.2 可用性
- Firebase SLAに準拠（99.95%）

### 12.3 スケーラビリティ
- Firebaseの自動スケーリングに依存

### 12.4 セキュリティ
- すべての通信はHTTPS
- Firebase Security Rulesによるアクセス制御
- パスワードはFirebase Authenticationで安全に管理
