# WorkOutApp プロジェクト固有ルール

## 問題対応フロー（必須・厳守）

問題やエラーが発生した際は、以下のフローを**必ず順番に**遵守すること。
**このフローを省略することは禁止。**

### フロー概要

```
問題発生 → TodoWrite追加 → 原因調査 → 修正計画 → problem.txt記録 → 修正実施 → 動作確認 → TodoWrite完了
```

### 1. TodoWrite にタスクを追加（最初に必ず実行）

問題が報告されたら、**修正作業を開始する前に**以下のタスクを TodoWrite に追加：

```
- [ ] Problem #X: 原因調査
- [ ] Problem #X: 修正計画作成
- [ ] Problem #X: problem.txt に記録
- [ ] Problem #X: 修正実施
- [ ] Problem #X: 動作確認
```

### 2. 原因調査
- 問題の根本原因を徹底的に調査する
- 表面的な対処ではなく、なぜ発生したかを特定する
- 類似の過去問題がないか problem.txt を確認する

### 3. 修正計画の作成
- 調査結果に基づき、具体的な修正計画を立てる
- 修正の優先順位を明確にする
- 修正するファイルを特定する

### 4. problem.txt への記録
- problem.txt のテンプレートに従って記録
- **必須チェックリスト**を含めること
- 記録なしで修正作業を開始することは**禁止**

### 5. 修正実行
- 計画に基づいて修正を実施
- 修正完了後、結果を problem.txt に追記

### 6. 動作確認
- 問題が解決したことを確認
- 副作用がないことを確認
- `npm run build` / `npx tsc --noEmit` でビルドが通ることを確認

### 7. TodoWrite タスク完了
- 全てのタスクを完了にする
- problem.txt のステータスを「解決済み」に変更
- 必須チェックリストを全てチェック

### 違反時の対応
- このフローを省略して修正を開始した場合、即座に作業を中断
- 最初からフローをやり直す

## Phase完了時のチェック

各Phase完了時に必ず実施：
1. `npm run build` でビルドが通ることを確認
2. `npx tsc --noEmit` で型チェックが通ることを確認
3. エラーがある場合は上記の問題対応フローを実行

## ルート定義のルール

- ページが実装されていない状態でルート定義を追加しない
- ルート追加時は対応するページ実装を同時に行う
